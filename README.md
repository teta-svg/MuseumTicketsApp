# Курсовой проект: Web-приложение для продажи билетов в музейные комплексы

## О проекте
Проект построен на базе **.NET 10 / C#** с использованием **Entity Framework Core** и принципов многослойной архитектуры (**Clean Architecture**).

---

## Структура решения
Решение разделено на логические слои для обеспечения чистоты и независимости кода:

### 1. Core (Ядро)
- **Museum.Domain** — Сущности предметной области.
- **Museum.Application** — Логика приложения:
  - DTOs.
  - Interfaces.
  - Services.

### 2. Infrastructure (Инфраструктура)
- **Museum.Persistence** — Слой доступа к данным:
  - `MuseumTicketsDBContext` с Fluent API конфигурациями.
  - Репозитории для работы с выставками, билетами и пользователями.
  - Управление миграциями базы данных.

### 3. Presentation (Представление)
- **Museum.BlazorUI** — **Единый запускаемый проект**:
  - Интерактивный интерфейс на базе компонентов Blazor.
  - Встроенный API-слой с поддержкой Swagger/OpenAPI.
  - Обработка публичных и защищённых эндпоинтов.

---

В проекте реализовано минимальное и эффективное взаимодействие с JavaScript для задач, выходящих за рамки возможностей C#:
- **Аутентификация (`window.auth`):** Управление жизненным циклом JWT-токена в `localStorage`.
- **Файловая система (`window.saveAsFile`):** Генерация и скачивание электронных билетов напрямую из браузера через Base64-кодирование.

---

## Технологический стек
- **Язык:** C# / .NET 10
- **Frontend:** Blazor (Interactive Mode)
- **JS Interop:** LocalStorage & File Download logic
- **ORM:** Entity Framework Core (Database First)
- **СУБД:** Microsoft SQL Server
- **Безопасность:** JWT (JSON Web Tokens) & ASP.NET Core Identity

---

## Быстрый старт
1. **Запуск:** Установить **`Museum.BlazorUI`** как стартовый проект и запустить.
2. **Swagger:** Документация API доступна по адресу `/swagger` в рамках запущенного приложения.
